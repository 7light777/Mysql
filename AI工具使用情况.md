# AI 工具使用情况说明

## 一、使用目的

在本次“篮球数据管理系统”课程项目开发过程中，我使用了 AI 工具作为辅助工具，主要是为了提高资料整理、页面设计构思、文档表达和问题排查的效率。项目的系统主题、数据库表结构理解、功能取舍、运行调试和最终结果确认都由我完成。

AI 工具的使用重点主要放在一些辅助性工作上，例如页面布局建议、报告语言润色、接口测试说明整理、错误日志解释和代码结构检查等。对于数据库课程要求中的关键内容，如事务删除、触发器添加、存储过程更新和视图查询，我还是按照课程要求自己理解、调试和验证，并结合项目实际功能完成最终确认。

## 二、主要使用范围

| 使用场景 | 使用方式 | 说明 |
| --- | --- | --- |
| 前端页面设计 | 辅助生成页面布局思路、配色建议和交互形式 | 参考管理系统常见风格，确定左侧导航栏、顶部栏、卡片式内容区、表格和表单布局 |
| 页面文案整理 | 辅助优化页面标题、说明文字和按钮提示 | 使页面更适合课程答辩展示和报告截图 |
| 文档润色 | 辅助调整报告语气和段落结构 | 将口语化表达改为更正式的课程报告语言 |
| 接口测试说明 | 辅助整理 API 测试步骤和演示顺序 | 便于后续使用浏览器或接口工具进行功能演示 |
| 错误信息理解 | 辅助分析 Maven、JDK、MySQL 连接和运行时报错 | 根据报错信息定位问题，例如 JDK 版本、数据库未创建、表名不一致等 |
| 代码自查 | 辅助检查项目结构、文件命名和说明是否一致 | 用于发现页面、接口、文档之间描述不一致的问题 |

## 三、前端设计辅助情况

在前端页面设计方面，AI 工具主要用于提供界面布局和视觉风格建议。本系统最终采用较为简洁的后台管理系统风格，包括深蓝色侧边栏、白色顶部导航栏、浅灰色页面背景、白色内容卡片、蓝色主按钮和红色危险按钮等。

页面设计时重点考虑课程演示需求，因此没有采用复杂前端框架，而是使用 Thymeleaf、HTML、CSS 和少量 JavaScript 完成页面展示。AI 工具在该部分主要用于辅助整理页面结构，例如首页仪表盘、球员档案、比赛管理、球探报告、数据分析和更新比赛数据等页面的内容分区。

最终页面内容和交互逻辑都是根据本项目已有后端接口和数据库功能调整的，主要目的是保证页面能够支撑课程报告截图和现场答辩演示。

## 四、文档和表达辅助情况

在课程报告和说明文档编写过程中，AI 工具主要用于辅助调整语言表达和组织结构。例如在描述事务、触发器、存储过程和视图功能时，采用“功能描述、涉及数据表、字段关系、实现方式、演示步骤、结果说明”的结构，使报告内容更加清晰。

对于报告中的关键技术说明，我会根据项目实际代码和运行结果进行核对，避免出现与系统实现不一致的描述。AI 工具只是作为文字表达和结构整理的辅助工具，不替代我对功能逻辑的理解。

## 五、调试和问题排查辅助情况

项目开发过程中出现过 JDK 版本不匹配、Spring Boot 启动失败、MySQL 数据库不存在、表名大小写或命名不一致、页面数据显示为空等问题。AI 工具用于辅助解释错误日志含义，并给出排查方向。

例如，当系统提示找不到数据库或表时，我会根据提示检查数据库初始化脚本和 MySQL 中的实际表结构；当 Maven 提示 JDK 版本问题时，我也会检查并调整本机 Java 版本配置。相关问题都通过本地运行、数据库导入和页面访问进行了验证。

## 六、使用边界说明

本项目没有把 AI 工具作为简单复制粘贴核心成果的来源。AI 输出内容在使用前都会经过我自己的阅读、筛选、修改和验证。

数据库设计、业务功能选择、接口演示流程和最终提交内容都以课程要求和我的实际调试结果为准。对于数据库脚本、后端接口和页面交互等内容，我也结合项目运行情况进行了检查，确保它们能够支撑课程答辩和报告截图。

## 七、总结

总体来看，AI 工具在本项目中主要承担辅助角色，帮助我提高页面设计、文档整理和问题排查效率。项目的核心功能实现、数据库对象理解、运行调试和最终结果确认都由我完成。通过合理使用 AI 工具，本项目在保持课程要求完整性的基础上，也提高了开发和整理材料的效率。
